1. Resumen Ejecutivo

Se desarrolló una aplicación web de gestión clínica (SPA) completa para el negocio de estética Mary Care, de la propietaria María Frías. La aplicación funciona como Progressive Web App (PWA), es decir, puede instalarse en el teléfono como si fuera una app nativa tanto en iPhone como en Android, sin necesidad de tienda de aplicaciones.

Toda la información se guarda de forma segura en la nube mediante Supabase, una base de datos PostgreSQL, lo que garantiza que los datos estén siempre disponibles desde cualquier dispositivo.

3. Funcionalidades Implementadas

3.1 Control de Inventario Automático

Nuevo

Al registrar una venta de productos, el sistema descuenta automáticamente las unidades vendidas del inventario del catálogo. También se puede registrar reabastecimiento manualmente desde el catálogo, sumando unidades al stock disponible.

  • Alerta visual cuando un producto tiene stock bajo o agotado
  • El margen de ganancia se calcula automáticamente (precio venta − costo)

3.2 Reporte de Sesión por WhatsApp

Nuevo

Desde el módulo de Sesiones, se puede enviar al cliente un reporte detallado de su visita directamente por WhatsApp. El mensaje incluye: tratamiento realizado, productos aplicados, evolución, notas y fecha de próxima cita.

El envío requiere autorización con PIN (0827) para evitar envíos accidentales.

3.3 Comprobante de Pago por WhatsApp

Nuevo

Desde el módulo de Movimientos, cada registro tiene un botón de WhatsApp que genera automáticamente un comprobante de pago formateado con: cliente, fecha, servicio/producto, método de pago y monto total. También requiere PIN de autorización.

3.4 Módulo de Gastos & Costos

Nuevo

Registro detallado de todos los gastos del negocio, clasificados por categoría: Productos e insumos, Local y servicios, Equipamiento, Marketing, Personal, Capacitación, Mantenimiento y Otros.

  • KPIs de margen neto e ingresos vs. gastos del mes actual
  • Gráfica de tendencia de los últimos 6 meses
  • Filtros por categoría y búsqueda por concepto

3.5 Módulo de Cotizaciones

Nuevo

Creación de presupuestos personalizados para clientes con múltiples ítems (servicios y productos). Cada cotización tiene fecha de validez, estado y cálculo automático del total.

  • Estados: Pendiente, Aprobada, Rechazada
  • Envío de cotización formateada por WhatsApp con PIN de autorización
  • Edición y eliminación de cotizaciones

3.6 Reporte Mensual

Nuevo

Nuevo tipo de reporte en Reportería con selector de mes y año. Muestra un análisis completo del período seleccionado:

  • Ingresos totales, Gastos totales, Margen neto y % de margen
  • Total de sesiones y clientes atendidos en el mes
  • Top 5 servicios más vendidos del mes
  • Top 5 clientes que más gastaron en el mes

3.7 Abonos Parciales en Deudas

Nuevo Mejorado

Sistema de pagos parciales para deudas por cobrar. Al registrar un abono:

  • Se selecciona el método de pago (Efectivo, Transferencia, Tarjeta)
  • Se muestra cuánto queda pendiente en tiempo real
  • Se crea automáticamente un movimiento de ingreso (el balance se actualiza)
  • Cuando el abono cubre el total, la deuda se marca automáticamente como pagada

4. Seguridad y Acceso

4.1 PIN de Acceso

La aplicación requiere un PIN de 4 dígitos para acceder. La sesión se mantiene activa mientras el navegador esté abierto; al cerrar el navegador se solicita el PIN nuevamente.

PIN actual: 0827 — Se recomienda no compartirlo. Está codificado directamente en la aplicación y puede cambiarse modificando el valor PIN_CODE en el código fuente.

4.2 PIN de Autorización para WhatsApp

El envío de mensajes por WhatsApp (reportes de sesión, comprobantes de pago y cotizaciones) requiere ingresar el mismo PIN de acceso como confirmación. Esto evita envíos accidentales a clientes.

4.3 Anti-Caché (PWA)

La aplicación incluye un sistema de versiones que fuerza la recarga automática cuando hay una actualización disponible. Esto es especialmente importante en iOS donde el navegador guarda versiones antiguas en caché. La versión actual es v18.

5. Base de Datos — Migraciones SQL

La base de datos es Supabase (PostgreSQL). Las siguientes migraciones deben ejecutarse en el SQL Editor del panel de Supabase para que todas las funciones estén operativas:

-- 1. Agregar costo e inventario al catálogo
ALTER TABLE catalogo ADD COLUMN IF NOT EXISTS costo numeric DEFAULT 0;
ALTER TABLE catalogo ADD COLUMN IF NOT EXISTS cantidad integer DEFAULT 0;

-- 2. Agregar hora a las sesiones
ALTER TABLE sesiones ADD COLUMN IF NOT EXISTS hora time;
ALTER TABLE sesiones ADD COLUMN IF NOT EXISTS proxima_hora time;

-- 3. Abonos parciales + método de pago en deudas
ALTER TABLE deudas ADD COLUMN IF NOT EXISTS monto_pagado numeric DEFAULT 0;
ALTER TABLE deudas ADD COLUMN IF NOT EXISTS metodo_abono text DEFAULT 'efectivo';

-- 4. Tabla de gastos
CREATE TABLE IF NOT EXISTS gastos (
  id bigint generated always as identity primary key,
  fecha date not null default current_date,
  concepto text not null,
  categoria text default 'Otros',
  monto numeric default 0,
  notas text,
  created_at timestamptz default now()
);
ALTER TABLE gastos DISABLE ROW LEVEL SECURITY;

-- 5. Tabla de cotizaciones
CREATE TABLE IF NOT EXISTS cotizaciones (
  id bigint generated always as identity primary key,
  fecha date not null default current_date,
  cliente text not null,
  items jsonb not null default '[]',
  total numeric default 0,
  notas text,
  validez date,
  estado text default 'pendiente',
  created_at timestamptz default now()
);
ALTER TABLE cotizaciones DISABLE ROW LEVEL SECURITY;
Nota importante: Supabase activa automáticamente la seguridad de filas (RLS) en todas las tablas nuevas. Si alguna función muestra el error "row-level security policy", ejecutar ALTER TABLE [nombre_tabla] DISABLE ROW LEVEL SECURITY; en el SQL Editor.
